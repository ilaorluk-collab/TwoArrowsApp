# TwoArrowsApp (Mobile) — AGENTS.md

## Project Overview
Мобильное приложение для сервиса спид-дейтинга TwoArrows (CupidSeaside). Написано на React Native с использованием фреймворка Expo.

## Tech Stack
- Expo ~54.0.0
- React Native 0.81.5
- React 19.1.0
- expo-router (навигация через файловую систему)
- expo-glass-effect, expo-image и другие библиотеки экосистемы Expo

## Constraints & Rules
- **Запуск**: Всегда используйте `npx expo start` или npm скрипты `npm run start`.
- **Нативный код**: Запрещено вносить изменения напрямую в папки `android` или `ios` (если они появятся). Вся конфигурация только через `app.json` или плагины Expo.
- **Стиль**: TypeScript, функциональные компоненты, хуки.

- **ПРАВИЛО СТОП-КРАНА**: НИКАКИХ действий (редактирование кода, изменение конфигов, запуск команд, удаление файлов) без предварительного четкого плана. Агент имеет право делать ТОЛЬКО ЧТЕНИЕ до получения явного разрешения "Да" от пользователя. Это правило приоритетнее любых других инициатив.
- **ЗАПРЕТ НА ПРЕЖДЕВРЕМЕННУЮ РАДОСТЬ**: Агенту категорически запрещено кричать "Ура, я всё починил!" до того, как пользователь физически протестировал фикс. Выдавать только сухие факты и инструкции к тесту.
- **МЕТОД ИСКЛЮЧЕНИЯ**: При любых странных багах UI (особенно с касаниями или клавиатурой) агент обязан сначала предложить "Ядерный тест" (изоляцию компонента до голого текста/инпута), прежде чем строить теории и переписывать сложную логику.
- **ЖЕСТКАЯ БЛОКИРОВКА ВЕРСИЙ (EXPO 54)**: Практика показала, что малейшее отклонение от этих версий ломает сборку и роняет симулятор. Категорически ЗАПРЕЩЕНО обновлять или менять эти версии в `package.json` без отдельного явного приказа:
  - `expo`: `~54.0.0`
  - `react-native`: `0.81.5`
  - `react`: `19.1.0`
  - `react-dom`: `19.1.0`
  - `expo-router`: `~6.0.24`
  Любые новые пакеты должны устанавливаться СТРОГО через `npx expo install <пакет>`, чтобы Expo сам подобрал версию под SDK 54. Никогда не использовать `npm install` для нативных библиотек.
